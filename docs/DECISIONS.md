# Decisions

## Decision log
- Use a text-first Rojo workflow instead of a Studio-only binary workflow.
- Keep MVP to one store map and one currency.
- Server owns round state, rewards, and dangerous events.
- Cosmetics come before strong power monetization.
- OpenClaw uses `main` as orchestrator plus worker agents for design, engineering, content, and QA.
- Sprint 6 visual direction is locked to **stylized retail horror**: an ordinary late-night supermarket built from clean modular forms, readable materials, and fluorescent lighting that becomes subtly wrong under pressure.
- Sprint 6 remains a visual identity and environment art foundation sprint only; it does not add gameplay systems, economy changes, combat, matchmaking, or map expansion.
- Sprint 6 critical-path polish order is locked to: 1) lobby / entrance, 2) checkout zone, 3) one hero aisle, 4) freezer section, 5) stockroom corner.
- Sprint 6 ships on a **Studio-first with optional imports** policy; imported assets may polish the result but cannot be required for sprint success.
- Sprint 6 must remove the obvious primitive graybox read from the five priority zones, HUD skin, and key event states while preserving the Sprint 5 Ready playable baseline.
- Blackout is a readable low-light emergency state, mimic is a localized uncanny cue, and round-end success / failure each get explicit distinct summary treatments.
