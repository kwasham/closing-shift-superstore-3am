# Sprint 7 Plan — Full-Store Art Rollout + Store Presence

## Sprint status
- Planned
- Baseline assumption: Sprint 6 is Ready and tagged
- This sprint must preserve the Sprint 6 playable baseline

## Sprint goal
Scale the accepted Sprint 6 visual language across the entire player-traversed store, remove the remaining obvious primitive look from gameplay-visible spaces, and refresh the public-facing asset pack so the game's store surface finally matches the in-game experience.

## Product thesis
Sprint 6 proved that the game can look intentional.
The next product risk is inconsistency: one polished slice beside still-placeholder spaces weakens first impression, trust, and screenshot value.
Sprint 7 should turn the visual slice into a believable full-store presentation without opening new gameplay scope.

## Production pillars
1. **Consistency over novelty**
   - Reuse the accepted visual language everywhere players actually go.
2. **Wayfinding is part of art**
   - Signs, category markers, checkout numbering, and staff-only cues should help both mood and navigation.
3. **Performance-aware rollout**
   - Scaling the art pass cannot quietly undo the stable baseline on phone or low-end clients.
4. **Public surfaces must be honest**
   - The icon and thumbnails should come from the real current game, not from a one-off mockup that players cannot recognize after joining.
5. **No systems churn**
   - Sprint 7 is not the moment for new mechanics, new maps, or economy changes.

## In scope
1. **Full-store visual rollout**
   - remaining aisles
   - remaining floor / wall / ceiling treatment
   - remaining checkout-adjacent space
   - connecting spaces and transitions
   - back-of-house spaces players can enter or see
   - lobby continuity pass so the store and lobby feel like one product

2. **Wayfinding and signage pass**
   - aisle numbering
   - department/category signage
   - checkout numbering
   - employee-only / hazard signs
   - emergency / blackout-readable markers
   - price-card / sale-card consistency rules

3. **Performance-aware cleanup**
   - remove or hide obsolete visible primitives
   - reduce duplicate decorative clutter where it hurts readability
   - keep lighting / decal / prop choices inside a defined budget
   - document any art-side exceptions that need runtime handling

4. **Store presence asset refresh**
   - final icon direction
   - thumbnail set
   - update / social shot set
   - capture checklist tied to the real live build

5. **Proof package**
   - before / after sweep of all player-visible zones
   - 1-player and 2-player runtime sanity after rollout
   - phone-size HUD / prompt readability
   - blackout and mimic readability in the now-expanded space

## Out of scope
- new mechanics or events
- new map
- new progression layers
- monetization changes
- full audio overhaul
- bespoke cinematic trailer production
- photoreal target
- deep back-of-house expansion beyond already playable / visible areas

## Required deliverables
- full-store replacement / rollout matrix
- signage and wayfinding rules
- performance budget and fallback rules
- updated store page asset pack direction
- full-zone proof capture set
- build / smoke / runtime sanity evidence

## Ready rule
Do not call Sprint 7 Ready unless QA can verify that:
- player-traversed spaces no longer feel half-polished and half-placeholder,
- wayfinding and signage are readable during normal and stressed states,
- phone HUD / prompt readability remain green,
- public-facing capture assets honestly match the shipped build,
- build / smoke / live runtime sanity stay green.

## Non-blocking follow-ups
- optional external-device spot checks beyond the accepted proof set
- extra decorative polish in non-playable background areas
- future seasonal or event reskins using the same kit
